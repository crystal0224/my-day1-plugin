# 정구봉님의 AI Native Camp를 비개발자 친화 버전으로 디벨롭한 이야기

## 시작: 완주하지 못하는 비개발자들

정구봉님의 **AI Native Camp - Day 1**은 정말 멋진 온보딩 스킬입니다. "슬라이드 없이 Claude가 직접 가르치는" 대화형 학습이라는 컨셉이 혁신적이었죠. 하지만 주변 비개발자 동료들에게 추천했을 때, 대부분 중간에 이탈했습니다.

"CLI가 뭔지 모르겠어요."
"3시간은 너무 길어요."
"명령어가 안 돼서 막혔는데 어떻게 해야 할지..."

원본의 철학은 훌륭했지만, **비개발자가 완주하기엔 장벽이 있었습니다.** 그래서 "완주율 50% → 80%"를 목표로 my-day1을 만들었습니다.

---

## 문제 분석: 왜 이탈할까?

실제 비개발자들이 막힌 지점을 관찰했습니다:

**1. 용어 장벽**
"CLI는 Command Line Interface입니다" → 여전히 모릅니다.
HR 담당자에게 "CLI"는 생소한 영어 약자일 뿐입니다.

**2. 시간 부담**
13개 블록, 3시간. 바쁜 현업에겐 큰 결심이 필요합니다.
"나중에 시간 날 때..." → 영원히 안 합니다.

**3. 막혔을 때 도움 없음**
"명령어를 입력했는데 안 돼요."
어디가 잘못됐는지, 어떻게 해야 하는지 알 수 없습니다.

**4. 동기부여 부족**
"왜 이걸 배워야 하지?" 내 업무와 어떤 관련이 있는지 모릅니다.

**5. 재진입 불가**
5개 완료 후 급한 일로 중단 → 3일 후 "어디까지 했더라?" → 포기

## 접근 방법: 3가지 핵심 전략

### 1️⃣ 직무별 맞춤 학습 - "내 언어로 설명해줘"

**문제**: 모두에게 같은 기술 용어 사용
**해결**: 직무별로 다른 비유 제공

예를 들어 "CLI" 설명:
- **HR**: "Excel 수식처럼, 텍스트로 컴퓨터에 명령하는 방법"
- **디자이너**: "Figma 단축키의 강력 버전"
- **PM**: "프로젝트 커맨드 센터"

**구현 방법**:
- 15개 기술 용어 × 10개 직업 = 150개 비유 작성
- 프로파일링: 처음에 직업/전공/터미널 경험 3가지 질문
- 동적 설명: 용어 출현 시 해당 직무 비유로 자동 변경

**차별점**: 원본은 "정의"를 알려주고, my-day1은 "비유"로 이해시킵니다.

### 2️⃣ Essential Path - "핵심만 빠르게"

**문제**: 13개 블록 3시간은 심리적 부담
**해결**: 7개만 완료해도 실무 투입 가능

**분류 전략**:
- **필수 7개** (1~1.5시간): Setup, Experience, CLAUDE.md, Skill, Subagent, 요약, Basics
- **심화 6개** (선택): Why, MCP, Agent Teams, Hook, Plugin, Break

**학습 경로 선택**:
1. 빠른 시작 (필수만)
2. 전체 과정 (13개)
3. 커스텀 (직접 선택)

**마일스톤 배지**:
- 🎯 기초 완료 (3개) → 성취감
- 🏆 핵심 완료 (7개) → "이제 쓸 수 있다!" 자신감
- 🚀 마스터 (13개) → 전문가

**차별점**: 원본은 "올 오어 낫씽", my-day1은 "점진적 성장"입니다.

### 3️⃣ 막힘 방지 시스템 - "도움이 필요해"

**문제**: 막혔을 때 어떻게 할지 모름
**해결**: Help 시스템 + 재진입 플로우

**Help 시스템 (4가지 상황)**:
```
help 입력 시:
1. 명령어가 안 돼요
   → 화면 캡처 요청 → 진단 → 구체적 해결법
2. 무슨 말인지 모르겠어요
   → 더 쉬운 비유로 재설명
3. 이 미션 건너뛰고 싶어요
   → skip 가능 (필수는 경고)
4. 다른 질문이 있어요
   → 자유 질문
```

**재진입 플로우**:
```
3일 후 재접속:
"👋 다시 오셨네요!"
진행 상황: 4/7 완료 (57%)
획득 배지: 🎯 기초 완료

선택:
1. 이어서 하기 (추천)
2. 처음부터 다시
3. 특정 미션 선택
```

**차별점**: 원본은 "혼자 헤쳐나가기", my-day1은 "언제든 도움 요청"입니다.

## 구현: Claude와 팀 협업

흥미로운 점은, **이 프로젝트 자체가 Claude Code의 Agent Teams 기능을 활용**했다는 것입니다.

**작업 과정**:
1. **Brainstorming**: 요구사항 명확화, 3가지 접근법 설계
2. **Writing Plans**: 12개 태스크로 구조화
3. **Subagent-Driven Development**: 8명의 에이전트 병렬 작업
   - motivation-system-dev: 동기부여 시스템
   - glossary-system-dev: 용어 비유 시스템
   - help-system-dev: Help 시스템
   - (총 8명이 동시 작업)

**결과**: 순차 작업 시 80분 → 병렬 작업 10분 = **8배 효율 향상**

메타적이게도, "Claude Code로 만드는 Claude Code 온보딩"이었습니다.

## 배운 점: 좋은 컨텐츠 vs 완주 가능한 컨텐츠

정구봉님의 원본은 **"좋은 컨텐츠"**였습니다.
my-day1은 **"완주 가능한 컨텐츠"**를 목표로 했습니다.

**좋은 컨텐츠**:
- 정확한 정의
- 체계적인 구조
- 완전한 커리큘럼

**완주 가능한 컨텐츠**:
- 이해 가능한 비유
- 심리적 부담 최소화
- 막힘 지점마다 도움

두 가지가 결합되어야 **비개발자도 끝까지 갈 수 있습니다.**
